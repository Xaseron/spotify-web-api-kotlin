(common, jvm) fun spotifyClientPkceApi(clientId: String?, redirectUri: String?, token: Token, codeVerifier: String, options: SpotifyApiOptionsBuilder? = null): SpotifyClientApiBuilder

Instantiate a new SpotifyClientApiBuilder using a token and codeVerifier. This is for PKCE authorization.

Use case: I am using the PKCE client authorization flow. I want access to both public and client Spotify API endpoints, I have an existing api token, and I might want to set api options without using the DSL.

Parameters

clientId - Spotify client id

redirectUri - Spotify redirect uri

token - Api token retrieved using PKCE authorization flow.

codeVerifier - Your code verifier plaintext.

options - Override default API options such as the cache limit

Return
Configurable SpotifyClientApiBuilder that, when built, creates a new SpotifyClientApi
